# F2P · Files to Picture

打包隐写不压缩。把任意文件直接写进 BMP 像素字节，塞图片里带走。

## 特性

- **打包隐写** 多个文件打包进一张 BMP，不压缩，数据原样嵌入像素
- **超大文件支持** 单文件上限 2^53 字节，文件数上限 2^32，总 payload 不限
- **流式处理** `FileReader` 分块 + Promise 链背压 + StreamSaver 直写磁盘，不 OOM
- **按需解码** 解析 BMP 时只读元信息，点哪个文件再按需提取
- **可调分块** 1 KB ~ 64 MB，大块利用磁盘 IO，小块兼容低端设备
- **编解码分离** 顶部标签切换，编码/解码互不干扰
- **纯前端** 所有操作在浏览器完成，不上传服务器

## 二进制格式

BMP 像素区从字节 0 开始：

```
[4B marker] [4B fileCount] [元信息区] [数据区]
```

marker = `F2P1`（`0x46325031`），用于识别格式版本。

元信息区按文件数重复：

```
[2B nameLen] [UTF-8 name] [8B dataLen]
```

全部大端。解码时先读 4 字节判断是否为 `F2P1`（兼容旧格式），然后读 4 字节文件数。文件大小用 8 字节（64 位），总 payload 通过累加元信息得出，不受 4GB 字段限制。

## 使用

```
npm run dev      # 开发服务器 → http://localhost:3000
npm run build    # 生产构建 → dist/
```

1. 打开页面，顶部选编码/解码模式
2. 编码：拖放文件 → 选分块大小 → 「生成图片」→ 浏览器弹出保存对话框
3. 解码：上传 BMP → 「提取」→ 点文件名下载单个文件

## 分块大小

读取分块在顶部栏设置，编解码共用：
- **大文件、大分块**（4 MB+）：磁盘 IO 吞吐高，适合高性能设备
- **小文件、小分块**（64 KB 以下）：兼容旧设备，内存占用低
- 默认 64 KB，如果磁盘 IO 够快可以调到 4~16 MB

## 构建

```bash
npm install
npm run build    # Rspack + html-minifier-terser 后处理
```

产物在 `dist/`，通过 Cloudflare Pages 或任意静态托管部署。
